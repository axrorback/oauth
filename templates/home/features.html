Coderboys ID tizimining ishlash mantiqi va afzalliklari

Bu qanday servis?

**Coderboys ID** — bu markazlashgan autentifikatsiya va avtorizatsiya provayderi. U "Identity as a Service" (IDaaS) modelida ishlaydi. Bu shuni anglatadiki, siz o'z loyihangizda foydalanuvchilarni ro'yxatga olish, parollarni saqlash va login qilish qismini yozib o'tirmaysiz. Barcha og'irlikni bizning serverlar ko'taradi.

OAuth2 vs Oddiy API Auth

Nega shunchaki Login API emas, aynan OAuth2? Farqini quyidagi jadvalda ko'rishingiz mumkin:

Xususiyat Oddiy API (JWT) Coderboys OAuth2
Xavfsizlik Parol sizning bazangizda turadi Parol bizda, sizda faqat Token
SSO imkoniyati Faqat bitta saytda ishlaydi Barcha loyihalaringizga bitta login
Uchinchi tomon Xavfli (parol berish kerak) Xavfsiz (faqat ruxsat beriladi)

Integratsiya Flow

Agar foydalanuvchi sizning saytingizda "Coderboys orqali kirish" tugmasini bossa, jarayon quyidagicha kechadi:

1
Redirect: Saytingiz foydalanuvchini bizning /authorize/ sahifamizga yuboradi.
2
Consent: Foydalanuvchi o'z login/parolini kiritadi va sizning ilovangizga ruxsat beradi.
3
Callback: Biz foydalanuvchini maxsus code bilan sizning saytingizga qaytaramiz.
4
Token Exchange: Sizning backend-ingiz ushbu kodni bizga yuborib, access_token oladi.
POST /o/token/

Foydalanuvchi va Dasturchi uchun foydasi

  • Vaqt tejalishi: Login/Register funksiyasini qurish uchun ketadigan 10-15 soatlik ishni 15 daqiqaga qisqartirasiz.
  • Ishonch: Foydalanuvchilar har bir yangi saytda yangi parol o'ylab topishdan charchashmaydi.
  • Xavfsizlik: Ma'lumotlar buzib kirilsa ham, sizning bazangizda foydalanuvchi parollari bo'lmaydi.
  • SSO: Barcha loyihalaringizga bitta login.
  • UX: Foydalanuvchilar faqatgina Sign with CODERBOYS orqali sizning ilovadan foydalanishi mumkin!
